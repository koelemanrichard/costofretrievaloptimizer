# Semantic SEO Content Audit Checklist

**Gebaseerd op Koray Tuğberk Gübür's Holistic SEO Framework**
**Versie 1.0 — Februari 2025**

Gebruik deze checklist om elk stuk content te beoordelen vóór publicatie. Elke regel is afgeleid uit de principes van Algorithmic Authorship, Micro-Semantics, Contextual Flow, Cost of Retrieval en EAV Architecture.

---

## Hoe te gebruiken

- ✅ = Regel correct toegepast
- ❌ = Overtreding gevonden — actie nodig
- ➖ = Niet van toepassing op dit contenttype

Score elke sectie apart. Streef naar **85+/100** totaalscore.

## II. ENTITY-ATTRIBUTE-VALUE (EAV) KWALITEIT

### A. Triple-structuur

| # | Regel | ✅/❌ |
|---|-------|-------|
| 15 | Alle feiten zijn gestructureerd als Subject-Predicate-Object (EAV triples) | |
| 16 | Elke zin levert bij voorkeur ÉÉN unieke EAV triple | |
| 17 | Entiteiten expliciet benoemd — geen ambigue voornaamwoorden ("het", "dit", "hij") waar verwarring mogelijk is | |
| 18 | Waarden specifiek en kwantitatief — geen "veel", "aanzienlijk", "diverse", "structureel lager" | |
| 19 | Getallen inclusief eenheden (€50, 15 werkdagen, 30 m², 9-13 jaar) | |

## SNELLE REFERENTIE — TOP 20 MEEST VOORKOMENDE FOUTEN

| # | Fout | Gevolg | Oplossing |
|---|------|--------|-----------|
| 1 | Ontbrekende canonical tag | Duplicate content risico | `<link rel="canonical" href="...">` toevoegen |
| 2 | Geen JSON-LD structured data | Geen rich results, lagere CTR | Schema.org markup toevoegen |
| 3 | `<figure>` in `<p>` | Onvoorspelbare DOM-rendering | `<p>` verwijderen rond `<figure>` |
| 4 | `<strong>` als pseudo-heading | Gebroken semantische structuur | Vervang door echte `<h4>` tag |
| 5 | Duplicate H1 en eerste H2 | Verwarrend voor extractie-algoritmes | H2 hernoemen naar specifiek subthema |
